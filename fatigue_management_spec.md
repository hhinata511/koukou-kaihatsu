# 陸上競技短距離選手向け疲労管理支援システム 仕様書

## 1. 概要

### 1.1 システム名
陸上競技短距離選手向け疲労管理支援システム

### 1.2 目的
短距離選手の日々の練習データ、睡眠情報、主観的疲労度、環境情報および動作解析結果を統合し、疲労状態を定量的に評価する。

### 1.3 開発環境
- フレームワーク：Flask
- 言語：Python
- フロントエンド：HTML / CSS / JavaScript
- データベース：SQLite（開発段階）
- 動画解析：OpenCV, MediaPipe（予定）

## 3.2 睡眠時間

### 初期設定
ユーザーが平均睡眠時間を登録する。

例：
- 平均睡眠時間 = 7時間30分

### 入力値
- 当日の睡眠時間

### 出力値
睡眠スコア

計算例

```text
睡眠差分 = 当日睡眠時間 − 平均睡眠時間
```

差分が大きいほど疲労への影響を考慮する。

# 4. 動作解析仕様

## 4.1 走行スコア算出

取得指標

- ストライド
- ピッチ
- 関節角度

疲労時に見られる特徴

- 接地時間増加
- 骨盤低下
- 膝が上がらない
- 上半身の力み

基準値との差分から走行スコアを算出する。

例

```text
走行スコア =
ストライド変化率 +
ピッチ変化率 +
関節角度変化率
```

---

# 5. 疲労スコア計算

## 5.1 日次疲労スコア

```text
疲労スコア =
練習強度スコア
+ 睡眠スコア
+ 主観的疲労度
+ 気温スコア
+ 走行スコア
```

---

## 5.2 練習強度スコア

練習メニューごとに事前設定する。

例

|メニュー|強度|
|---|---|
|流し|1|
|100m×3|3|
|150m×5|5|
|300m×4|7|
|サーキット|8|

---

## 5.3 週間スコア

```text
週間スコア =
1週間の日次疲労スコア合計
```

---

## 5.4 リスク判定

例

|週間スコア|状態|
|---|---|
|0～49|安全|
|50～79|注意|
|80以上|危険|

※閾値は今後のデータ収集によって調整する。
