# 陸上競技短距離選手向け疲労管理支援システム - 利用マニュアル

## 1. はじめに

本システムは、短距離選手の日々の練習負荷、睡眠状態、主観的疲労度、気温を統合し、**日次疲労スコア** および **週間疲労スコア** を自動算出することで、疲労蓄積の見える化と練習調整の支援を行います。

---

## 2. 起動方法

### 2.1 初回セットアップ

```bash
# 依存パッケージのインストール
pip install flask flask-sqlalchemy werkzeug

# アプリケーションのディレクトリへ移動
cd fatigue_app
```

### 2.2 アプリケーション起動

```bash
python app.py
```

起動後、ブラウザで以下のURLにアクセスしてください。

```
http://127.0.0.1:5000
```

アプリケーションを停止するには、ターミナルで `Ctrl + C` を押してください。

## 3. 画面と機能の説明

### 3.1 トップ画面（ダッシュボード）

URL: `/`

システム起動後に最初に表示される画面です。以下の情報が一目で確認できます。

| 表示項目 | 説明 |
|---|---|
| **本日の疲労スコア** | 今日入力された記録に基づく疲労スコア。未入力の場合は「-」と表示されます |
| **週間疲労スコア** | 直近7日間の日次疲労スコアの合計値です |
| **リスク判定** | 週間スコアに基づく3段階判定（安全 / 注意 / 危険） |
| **スコア内訳** | 本日の記録がある場合、練習強度・睡眠・主観疲労度・気温の各スコア内訳を表示 |
| **直近の記録** | 最新7件の記録一覧 |

#### リスク判定の見方

| 表示 | 週間スコア（初期値） | 意味 |
|---|---|---|
| 🟢 **安全** | 0 ～ 49 | 疲労は許容範囲内です。通常通りの練習を継続できます |
| 🟡 **注意** | 50 ～ 79 | 疲労が蓄積しています。練習強度の調整を検討してください |
| 🔴 **危険** | 80 以上 | 疲労が危険レベルです。練習量の大幅な削減または休息を推奨します |

> ※ 閾値は設定画面から変更可能です。

---

### 3.2 記録入力画面

URL: `/record/new`

日々の練習データを入力する画面です。

#### 入力項目

| 項目 | 入力方法 | 備考 |
|---|---|---|
| **日付** | 日付選択 | 必須。初期値は本日日付 |
| **練習メニュー** | セレクトボックス | 必須。メニュー名と基礎点が表示されます |
| **本数** | 数値入力 | 0以上の整数。本数が多いほど練習強度スコアが増加します |
| **睡眠時間** | 数値入力（小数可） | 0～24時間。例: 7.5（7時間30分） |
| **主観的疲労度** | ラジオボタン（1～5） | 1=全く疲れていない / 5=かなり疲れている |
| **気温** | 数値入力 | ℃。25℃以上で疲労スコアに加算されます |
| **備考** | 自由入力 | 練習の感触など自由に記入できます |

#### 入力の流れ

1. 日付を確認・選択します
2. 実施した練習メニューを選択します
3. 本数を入力します（例: 100m×3本 → 「3」）
4. 前夜の睡眠時間を入力します
5. 現在の主観的疲労度を1～5で選択します
6. 練習時の気温を入力します
7. 必要に応じて備考を記入します
8. 「保存する」ボタンをクリックします

保存後、自動的に記録詳細画面へ遷移します。

2. 動画ファイルを選択（mp4, mov, avi, webm, mkv 形式）
3. 「アップロード」ボタンをクリック

> ※ MVP（現バージョン）では動画の保存のみ対応しており、自動動作解析は未実装です。今後のアップデートで追加予定です。

> ※ MVP（現バージョン）では動画アップロードと再生のみ対応しており、自動解析エンジンは **Phase 2/3** で実装予定です。

#### 3.6.1 画面の構成

動作解析画面は以下のブロックで構成されています。

| ブロック | 内容 |
|---|---|
| **記録サマリー** | 対象の記録日・練習メニュー・日次疲労スコアを表示 |
| **アップロード済み動画一覧** | 記録に紐づく全動画をリスト表示 |
| **動画プレビュー** | ブラウザ上で動画を再生可能 |
| **解析結果** | 解析済みの場合、各指標の数値と変化率を表形式で表示 |
| **動作解析の説明** | 将来的に抽出予定の指標と疲労との関連を解説 |

#### 3.6.2 動画のアップロード手順

動画は **記録詳細画面**（`/record/<id>`）からアップロードします。

1. 記録詳細画面下部の「動画アップロード」セクションに移動
2. 「動画タイプ」を選択
   - **走行映像**: 真横から撮影した走行動画（主に使用）
   - **垂直ジャンプ**: 滞空時間・ジャンプ高推定用（将来対応）
3. 動画ファイルを選択（対応形式: `.mp4`, `.mov`, `.avi`, `.webm`, `.mkv`）
4. 「アップロード」ボタンをクリック
5. アップロード完了後、動作解析画面で動画が確認可能になります

> 📌 **推奨撮影条件**
> - **画角**: 真横から全身が映るように固定カメラで撮影
> - **フレームレート**: 30fps 以上推奨（低フレームレートだと関節角度の抽出精度が低下します）
> - **解像度**: 1280×720 以上
> - **距離**: 走路全体または 30m〜50m 区間が収まる距離

#### 3.6.3 解析状態について

動画ごとに解析状態がステータスバッジで表示されます。

| ステータス | 表示 | 説明 |
|---|---|---|
| **未解析** | 🟡 未解析 | アップロード直後の初期状態。解析エンジン実装後に自動または手動で解析が実行されます |
| **解析済み** | 🟢 解析済み | 解析が完了し、各指標の数値が表示されます（Phase 2以降） |
| **失敗** | 🔴 失敗 | 解析中にエラーが発生した状態。動画形式や画角の不備などが原因として考えられます |

#### 3.6.4 将来の解析指標（Phase 2/3 実装予定）

本システムでは、OpenCV と MediaPipe を用いて以下の指標を動画から自動抽出する予定です。

##### ① ストライド（歩幅）

```
ストライド (m) = 走行距離 ÷ 歩数
```

- **目的**: 一歩あたりの移動距離を測定
- **疲労時の特徴**: ストライドが短くなる（股関節の可動域低下、出力減）
- **変化率の目安**: 基準値から 5% 以上の低下で要注意

##### ② ピッチ（歩数/秒）

```
ピッチ (steps/s) = 歩数 ÷ 時間
```

- **目的**: 単位時間あたりの足の回転数を測定
- **疲労時の特徴**: ピッチが低下する、または逆に無理に上げようとして上半身が力む
- **変化率の目安**: 基準値から 10% 以上の変動で要注意

##### ③ 股関節角度

- **測定方法**: 股関節（大転子）−膝関節−足首のなす角度を MediaPipe の姿勢推定ランドマークから算出
- **疲労時の特徴**: 股関節の屈曲角度が減少し、腿が上がらなくなる
- **評価基準**: 基準値との角度差分（°）を計測

##### ④ 膝関節角度

- **測定方法**: 膝関節の屈曲・伸展角度をランドマークから算出
- **疲労時の特徴**: 接地時の膝の沈み込みが大きくなる、または伸展が不十分になる
- **評価基準**: 基準値との角度差分（°）を計測

##### ⑤ 接地時間

- **測定方法**: 足部が地面に接触しているフレーム数をカウント
- **疲労時の特徴**: 接地時間が延長する（反発力の低下）
- **評価基準**: 基準値からの増加率（%）を計測

##### ⑥ 骨盤の上下動

- **測定方法**: 骨盤ランドマークの Y 座標の変動幅を計測
- **疲労時の特徴**: 骨盤が落ちる、上下動が大きくなる（体幹の安定性低下）
- **評価基準**: 基準値からの変動幅の増加率（%）を計測

#### 3.6.5 走行スコアの算出方法（将来）

動画解析結果から走行スコアを算出する予定の計算式です。

```
走行スコア = |ストライド変化率| + |ピッチ変化率| + |股関節角度変化率| + |膝関節角度変化率|
```

| 計算要素 | 内容 |
|---|---|
| **ストライド変化率** | (基準ストライド − 当日ストライド) ÷ 基準ストライド × 100 の絶対値 |
| **ピッチ変化率** | (基準ピッチ − 当日ピッチ) ÷ 基準ピッチ × 100 の絶対値 |
| **股関節角度変化率** | (基準角度 − 当日角度) ÷ 基準角度 × 100 の絶対値 |
| **膝関節角度変化率** | (基準角度 − 当日角度) ÷ 基準角度 × 100 の絶対値 |

> **基準値について**  
> システムでは、選手のコンディションが良い日の走行動画を「基準動画」として登録し、そこから抽出した各指標を基準値とします。  
> 日々の練習動画をこの基準値と比較することで、疲労によるフォーム変化を数値化します。

#### 3.6.6 解析ワークフロー（Phase 2/3 の予定）

```text
1. 動画アップロード（記録詳細画面）
       ↓
2. 動画ファイルをサーバーに保存
       ↓
3. 解析ジョブをキューに投入（非同期処理）
       ↓
4. OpenCV で動画をフレーム分割（30fps → フレーム画像）
       ↓
5. MediaPipe Pose で各フレームの姿勢ランドマークを検出
       ↓
6. ランドマーク座標から関節角度・ストライド・ピッチを算出
       ↓
7. 基準動画の値と比較し、変化率を計算
       ↓
8. 走行スコアを算出し、日次疲労スコアに合算
       ↓
9. 解析結果を JSON 形式で DB に保存
       ↓
10. 動作解析画面に結果を表示
```

#### 3.6.7 垂直ジャンプ解析（将来構想）

走行映像に加えて、垂直ジャンプ動画からの疲労評価も将来的に検討されています。

- **目的**: 下肢の瞬発力・反発力を評価
- **取得項目**: 滞空時間、推定ジャンプ高
- **疲労時の特徴**: ジャンプ高が低下する（神経筋疲労の指標）
- **撮影条件**: 真横から全身が映るよう固定カメラで撮影

```
推定ジャンプ高 (m) = (重力加速度 × 滞空時間²) ÷ 8
```

## 4. スコア計算の仕組み

### 4.1 計算式

```
日次疲労スコア = 練習強度スコア + 睡眠スコア + 主観的疲労スコア + 気温スコア + 走行スコア
```

### 4.2 各スコアの計算方法

#### 練習強度スコア

- 練習メニューごとに設定された**基礎点** × **本数係数** で算出
- 本数係数: 1本=1.0倍 / 2〜3本=1.2倍 / 4〜5本=1.5倍 / 6本以上=2.0倍

例: `100m × 3本` → 基礎点5 × 1.2 = **6.0**

#### 睡眠スコア

```
睡眠スコア = max(0, (平均睡眠時間 - 当日睡眠時間) × 2)
```

- 平均より多く寝た場合は 0（マイナスにはならない）
- 睡眠不足が大きいほど高スコア（= 疲労大）

例: 平均7.5時間 → 当日6.0時間 → (7.5 - 6.0) × 2 = **3.0**

#### 主観的疲労スコア

```
主観的疲労スコア = 入力値(1〜5) × 2
```

例: 「4: 疲れている」→ 4 × 2 = **8.0**

#### 気温スコア

```
気温スコア = max(0, (気温 - 25) × 0.5)
```

- 25℃未満はスコア0
- 高温環境ほど疲労に影響

例: 30℃ → (30 - 25) × 0.5 = **2.5**

#### 週間疲労スコア

```
週間スコア = 直近7日分の日次疲労スコアの合計
```

## 5. 利用シーン例

### ケース1: 通常練習日の記録

1. 練習後、記録入力画面を開く
2. メニュー「150m」、本数「3」、睡眠「7.0」、疲労度「3（普通）」、気温「22」を入力
3. 保存 → スコアを確認
   - 練習強度: 7.2 / 睡眠: 1.0 / 主観疲労: 6.0 / 気温: 0
   - 日次スコア: **14.2**

### ケース2: 高強度練習＋睡眠不足

1. メニュー「300m」、本数「4」、睡眠「5.0」、疲労度「5（かなり疲れている）」、気温「30」
2. 保存 → スコアを確認
   - 練習強度: 12.0 / 睡眠: 5.0 / 主観疲労: 10.0 / 気温: 2.5
   - 日次スコア: **29.5**

### ケース3: 週間の疲労蓄積チェック

1. トップ画面を表示
2. 週間疲労スコアを確認（例: 合計 72.3）
3. リスク判定「注意」→ 指導者と相談して翌週の練習メニューを調整

## 6. 注意事項

- 本システムは疲労状態の**参考指標**を提供するものであり、医学的診断を行うものではありません
- スコアや閾値は研究データに基づいて随時調整されることを前提とした仮設定です
- 動画の自動解析機能は**MVP完了後のPhase 2/3**で実装予定です
- データベースは開発段階のためSQLiteを使用しています（本番運用時はPostgreSQL等への移行を推奨）
- 記録の削除機能は管理画面で別途対応予定です
